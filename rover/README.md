# rover — автономная связка «вижу → еду → собираю → выезжаю»

Поведенческий «мозг» лунохода поверх детекции `navigation.py`. Логика написана
сразу под подключение моторов: всё железо спрятано за интерфейсами
(`hardware.py`), поэтому один и тот же автомат работает и в симуляции, и на
Raspberry Pi + STM32.

## Запуск симуляции (без железа)

```bash
python3 -m rover.app          # прогон демосцены: найти и собрать образцы
python3 tests/test_mission.py # тесты автомата, поиска, UART, порога
```

## Конечный автомат миссии (`mission.py`)

```
        ┌──────── граница/препятствие у носа ─────────┐
        ▼                                             │
     AVOID ──безопасно──► SEARCH ──цель видна──► APPROACH ──близко+по центру──► COLLECT
                            ▲  ▲                    │                              │
                            │  └──потеря цели────────┘                             │
                            └───────────── DEPART ◄──собрано, отъехать─────────────┘
```

| Состояние | Что делает |
|-----------|-----------|
| `SEARCH`   | Цели нет → систематический поиск (`search.py`). |
| `APPROACH` | Цель видна → пропорциональный подъезд, удерживая её в центре кадра; объезд препятствия по дальномерам. |
| `COLLECT`  | Цель близко и по центру → стоп, опустить-схватить-поднять; засчитывается только подтверждённый захват. |
| `DEPART`   | Отъезд назад и разворот от точки сбора → снова поиск. |
| `AVOID`    | Высший приоритет: тёмная граница или препятствие вплотную → отъезд и уход в свободную сторону. |

Приоритет: безопасность (`AVOID`) → незавершённая секвенция (`COLLECT`/`DEPART`)
→ цель (`COLLECT`/`APPROACH`) → поиск (`SEARCH`).

## Поиск (`search.py`)

Когда цели не видно — не крутимся на месте и не мечемся случайно:

1. **REACQUIRE** — доворот туда, где цель видели в последний раз (быстрее всего вернуть контакт).
2. **SCAN** — полный круговой обзор на месте (360°), чтобы детектор всё рассмотрел.
3. **RELOCATE** — переезд на новую точку обзора по расширяющейся спирали, с объездом
   препятствий по дальномерам; на новой точке снова обзор.

Накопленный поворот меряется по реальной одометрии; пройденные клетки
запоминаются, поэтому поиск расширяется наружу, а не топчется.

## Железо за интерфейсами (`hardware.py`)

| Интерфейс | Назначение | Симуляция | Реальное железо |
|-----------|-----------|-----------|-----------------|
| `Drive` | привод (2 мотора, дифференциал) | `SimDrive` | `uart_link.Stm32Link` |
| `Manipulator` | захват образца | `SimManipulator` | `uart_link.Stm32Link` |
| `TargetSensor` | цель в кадре | `SimTargetSensor` | камера+YOLO → `perception.build_target_view` |
| `RangeSensor` | препятствия (УЗ/ToF) | `SimRangeSensor` | дальномеры по UART/GPIO |
| `BoundarySensor` | тёмная граница под носом | `SimBoundarySensor` | `track.boundary_level` по кадру |

`DriveCommand(linear, angular)` нормирован в `[-1, 1]`; `to_differential()`
переводит его в скорости левого/правого колеса.

## Протокол UART к STM32 (`uart_link.py`)

Кадр: `0xAA  CMD  LEN  payload[LEN]  CRC8` (CRC-8, полином `0x07`, по `CMD+LEN+payload`).

| CMD | Имя | payload |
|-----|-----|---------|
| `0x01` | DRIVE | `int8 left, int8 right` (проценты −100..100) |
| `0x02` | MANIP | `uint8 action` (0=release 1=grip 2=lower 3=lift) |
| `0x03` | STOP | — |
| `0x10` | PING | — |

Питон-сторона готова; прошивке STM32 (`uart_version_1`) предстоит разобрать эти
кадры и крутить моторы/манипулятор — сейчас там loopback.

## Подключение реального лунохода

В `RoverMission` подставляются реализации интерфейсов:

```python
link = Stm32Link("/dev/serial0", 9600)     # привод + манипулятор по UART
mission = RoverMission(
    drive=link, manipulator=link,
    target_sensor=YoloTargetSensor(...),    # camera + ultralytics + perception
    range_sensor=...,                        # дальномеры
    boundary_sensor=...,                     # boundary_level по кадру
)
mission.run()
```

Поведенческий слой при этом не меняется — он не знает, едет он по-настоящему
или в симуляции.
